These compounding issues have left millions struggling to navigate airports independently. From long wait times for assistance to physical barriers in terminals, the lack of scalable solutions leaves many with mobility challenges behind.

Our Vision

We envision a future where mobility challenges are no longer barriers to travel. SmoothOperator is our response — a semi-autonomous robotic assistant designed to help travelers transport their luggage safely, independently, and with dignity.

Equipped with intelligent navigation, obstacle detection, and an easy-to-use interface, our robot transforms the travel experience for individuals with limited mobility, elderly passengers, and anyone overwhelmed by today’s under-resourced airport environments.

SmoothOperator supports independent navigation and lifting for all.

System Design

Architecture Overview

SmoothOperator is a high-performance autonomous luggage assistant engineered with a modular, yet tightly synchronized system architecture. The robot operates entirely untethered and autonomously, combining state-of-the-art robotics frameworks, embedded safety layers, and intelligent user interaction. Every subsystem—mechanical, computational, and user-facing—was designed, built, and integrated from the ground up to meet the challenges of real-world deployment in dynamic public environments.

Subsystems Overview

  • Advanced Navigation Stack: 11-node ROS system with SLAM, AMCL, and move_base.
  • Sensor Suite: LiDAR, IMU, and encoder fusion for real-time localization and costmap updates.
  • Mechanical Platform: Modular 80/20 chassis, high-torque dual-drive with 10:1 gear reduction, and a luggage forklift.
  • Embedded Safety: Tri-processor design (Jetson, Arduino Mega, Uno) with hardware overrides via ultrasonic and limit switches.
  • Communication Network: UART serial at 9600 baud and full-duplex data exchange via Node.js server APIs.
  • Human-Centric Interface: 21.5” touchscreen with animated face, voice output, and QR-based autonomous routing.
  • Remote Control: React Native app with secure pairing and full manual override capabilities.
  • Power System: Dual 12V 12Ah batteries with fused distribution and system-wide telemetry.

Implementation & Results

Mechanical Construction

Lightweight 80/20 T-slot aluminum frame with 10:1 gear train, high-torque DC motors, and impact-absorbing bumpers with limit switches for collision protection.

Exploded view of chassis and drive system

Technical Specifications

  • Frame: 80/20 T-slot aluminum (1" profile)
  • Drive: 10:1 gear reduction with 6" rubber wheels
  • Motors: 12V DC with 285 RPM, 125 oz-in torque
  • Lift System: Linear actuator with 80lb capacity
  • Safety: Force-sensitive bumpers with limit switches

The chassis design prioritizes stability during movement and lifting operations while maintaining a compact footprint for navigating narrow spaces in airport terminals.

Sensor Suite

360° RPLiDAR A1, ultrasonic sensors for low obstacles, high-precision wheel encoders, IMU for odometry, and camera for QR code scanning.

Sensor positioning and coverage zones

Sensor Specifications

  • 2D LiDAR: RPLiDAR A1 (360°, 12m range, 8000 samples/sec)
  • Ultrasonic: 3× HC-SR04 sensors (2-400cm range)
  • Through Bore Encoders: Quadrature encoders with 8192 CPR resolution
  • IMU: MPU6050 6-axis accelerometer and gyroscope
  • HD Camera: Logitech C270 HD (720p) with custom QR code algorithm
  • N-Open Limit Switch: Normally Open switches enabling for robust bumper detection

Our sensor fusion approach combines data from all sensors to create reliable environmental mapping and obstacle detection, with redundant systems for safety-critical functions.

Advanced Navigation System

ROS-powered navigation with AMCL localization, Dijkstra global planning, and DWA local planning for real-time path following and obstacle avoidance.

ROS navigation stack with real-time path planning

Navigation Architecture

  • Localization: Adaptive Monte Carlo Localization (AMCL)
  • Global Planner: Dijkstra algorithm with cost maps
  • Local Planner: Dynamic Window Approach (DWA)
  • Recovery: Custom behaviors for handling edge cases
  • Command Interface: Serial communication to motor controllers

The navigation system pre-loads terminal maps and can dynamically update them based on detected obstacles, allowing for efficient path planning even in crowded environments.

Human-Centric Interface

21.5" touchscreen with Kivy-based UI featuring facial animations and boarding pass scanning, plus a React Native mobile app for remote control.

Touchscreen and mobile app interfaces

Interface Features

  • Display: 21.5" 1080p touchscreen with anti-glare coating
  • Onboard UI: Python/Kivy framework with animated assistant
  • Mobile App: React Native with Bluetooth Low Energy pairing
  • Boarding Pass: QR code scanner with airline API integration
  • Accessibility: High-contrast mode, voice commands, multilingual support

Our dual-interface approach allows users to interact directly with the robot or remotely via smartphone, with a 4-digit pairing code ensuring secure connections and privacy protection.

Electronics Architecture

Dual 12V batteries, Jetson Nano for high-level processing, Arduino controllers for sensors and motors, all networked via USB/UART and Wi-Fi.

System architecture and power distribution

Electronics Specifications

  • Power: 2× 12V 12Ah lead-acid batteries with integrated BMS
  • Main Computer: Nvidia Jetson Nano (4GB RAM, 128GB SSD)
  • Microcontrollers: Arduino Mega 2560 for motors, Nano for sensors
  • Motor Drivers: Dual BTS7960 43A H-Bridge controllers
  • Networking: Integrated Wi-Fi 5 and Bluetooth 5.0 connectivity

The dual-battery system can hot-swap without interruption, providing extended runtime and redundancy. The segregated computing architecture isolates critical safety systems from higher-level functions.

Embedded Safety

Multiple safeguards including hardware bumpers, ultrasonic detection, radio e-stop, and watchdog processes for fail-safe operation.

Layered safety architecture and emergency systems

Safety Features

  • Physical: 360° soft bumpers with limit switch triggers
  • Sensing: Redundant ultrasonic blind-spot detection
  • Remote: 433MHz wireless emergency stop with 50m range
  • Software: Multi-level watchdog timers and self-diagnostics
  • Compliance: Meets ISO 13482:2014 personal care robot standards

Our safety-first design philosophy implements multiple independent systems that trigger motor cutoff when obstacles are detected. The robot automatically enters a safe state upon any component failure, protecting users and bystanders.

Version 0: Initial Proof-of-Concept

Our first prototype focused on basic locomotion, remote control, and structural integrity. Built using a simple chassis and manual control system, V0 served as a functional feasibility test of motor capabilities, wheelbase stability, and weight-bearing performance.

  • Custom-cut 80/20 aluminum frame
  • Basic dual-motor drivetrain
  • Manual lift operation
  • Arduino-controlled with joystick input

Watch our V0 proof-of-concept in action.

Final Prototype: Fully Integrated Smart Assistant

The final SmoothOperator system integrated hardware and software subsystems, enabling semi-autonomous operation. The robot can scan boarding passes, plan routes, lift luggage, and navigate terminals while ensuring user safety.

  • SLAM-based localization and navigation
  • Touchscreen-based interface and passcode pairing
  • Linear actuator with safe overload lift design
  • ROS-powered control and custom Android app
